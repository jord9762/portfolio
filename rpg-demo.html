Retro 90's Game

Project Overview

This 2D RPG is a passion project that pays homage to classic 90's Japanese role-playing games while incorporating modern programming techniques and unique features. Built entirely in Python using the Pygame framework, the game demonstrates my ability to architect complex systems and create engaging user experiences.

Key Features:

  • Turn-Based Combat System: A comprehensive combat engine with state management for different action phases, damage calculation, and AI decision-making.
  • Dynamic Animation: Frame-by-frame animation system for characters, spells, and UI elements that responds to game context and player actions.
  • Database Integration: SQLite backend that manages game items, character progression, and story elements with relational integrity.
  • Strava Integration: A unique feature that connects real-world fitness metrics from Strava to in-game character attributes, encouraging physical activity.
  • Physics System: Custom-built physics for world interaction, collision detection, and environmental effects.
  • Asset Pipeline: Complete workflow for creating, importing, and managing game assets including sprites, sound effects, and music.

Technical Implementation Details

Advanced Animation System

A sophisticated animation framework that manages sprite sheet rendering based on character state. The system handles different animation types (idle, walk, attack), frame rates, and contextual animations based on character health/status.

Dynamic UI Elements

Custom-built animation system for health and status bars that responds to game state changes. Includes visual cues like flashing yellow when limit gauge is full and red flashing for low health situations to add to suspense.

Floating Combat Text

A timing-based system for displaying damage numbers that provides clear feedback without overwhelming the player. Includes cubic easing functions for smooth animation and priority handling for simultaneous events i.e, multiple hits.

Custom UI Framework

Detailed implementation of UI elements that maintains the aesthetic of classic 90's JRPGs while providing modern usability.

Multi-layer Animation Coordination

Complex animation sequencing system that synchronizes player casting, spell effects, and enemy damage reactions. Uses a state machine to coordinate the timing of visual effects with gameplay mechanics.

SQL Database Integration

Relational database structure that powers the in-game shop and equipment systems. Uses foreign key relationships to connect items to character stats and progression, enabling dynamic pricing and item availability.

Comprehensive State Management

State machine architecture that handles different game phases such as attack sequences, ability activation, hit registration, and damage calculation. Provides a robust framework for complex turn-based combat mechanics.

Technical Architecture:

The game's architecture follows object-oriented design principles with careful separation of concerns between game logic, rendering, input handling, and data management. The codebase demonstrates advanced Python concepts including:

  • Class inheritance for creating specialized game entities
  • State pattern for managing complex gameplay transitions
  • Observer pattern for event handling and UI updates
  • SQL integration with proper transaction management
  • Asset management with efficient resource loading
  • Frame rate independence for consistent gameplay across different hardware
