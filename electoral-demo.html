Electoral Roll Text Recognition Tool

Confidentiality Notice: Due to the sensitive nature of electoral data, some elements of this project have been simplified or omitted to maintain confidentiality. The examples shown retain the technical aspects while protecting specific political data.

Project Overview

The Electoral Roll Text Recognition Tool is an innovative solution I developed for a political party to digitize and analyze electoral records. Traditional manual processing took up to 12.5 hours per document, but this automated solution reduced that time to just 18 minutes, representing a 97% efficiency improvement.

Key Features:

  • PDF Scanning with OCR: Utilizes Tesseract OCR to extract voting patterns from scanned PDFs, even with inconsistent markings.
  • Interactive Data Editor: Built with Flask to provide a user-friendly interface for verifying and correcting extracted data.
  • Geocoding Integration: Maps voter data geographically to identify patterns by location and generate accurate addresses for targeted communications.
  • Data Analysis: Provides visualization tools to help campaign managers identify voting trends and participation patterns.
  • Standalone Executable: Packaged as a simple executable that non-technical staff can use without Python installation or dependencies.

Technical Implementation Details

Document Analysis Challenge

The application processes documents with no clear structure and inconsistent metrics. I developed a robust OCR solution using Tesseract and Fitz to handle these variations while maintaining extraction accuracy.

Manual Record Addition

Users can add records directly through the interface when needed, maintaining database integrity while accommodating edge cases where automatic extraction might be insufficient.

Geocoding & Spatial Analysis

The application maps voter data geographically, allowing clients to understand voting trends by area and inform door-knocking strategies to target swing voters in key constituencies.

Advanced Filtering System

Custom filtering logic enables users to segment voter data by multiple criteria, such as highlighting citizens who haven't voted yet, facilitating targeted engagement campaigns.

Technical Requirements

Comprehensive documentation outlines the project's architecture and requirements, ensuring transparency in the solution's design and facilitating future enhancements if needed.

Interactive Voter Mapping

The webmap solution visualizes voter turnout patterns using clustered markers for confidentiality at the overview level, with the ability to drill down for more detailed analysis.

Technologies Used:

Python Flask Tesseract OCR Fitz/PyMuPDF Pillow PDFPlumber Folium Pandas NumPy Geocoding API SQLite

This project demonstrates my ability to combine computer vision, web development, and geospatial technologies to create practical solutions that deliver significant time savings and valuable insights while handling sensitive data with appropriate confidentiality measures.
